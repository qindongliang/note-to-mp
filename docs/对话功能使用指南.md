# 对话功能使用指南

## 概述

对话功能是 NoteToMP 插件新增的特色功能，可以创建类似微信聊天的对话界面。通过简单的 Markdown 语法，即可生成美观的问答、访谈、客服对话等内容。

## 基本语法

### 语法结构

```markdown
:::dialogue[对话标题]
角色名: 对话内容
角色名: 对话内容
...
:::
```

### 关键特性

- ✅ **可选标题**：`:::dialogue[标题]` 或 `:::dialogue`
- ✅ **灵活角色**：支持中英文角色名称
- ✅ **中英文冒号**：支持 `:` 和 `：` 两种冒号
- ✅ **自动交替**：消息自动左右交替显示
- ✅ **多行内容**：支持单行或多行对话内容

## 使用示例

### 示例1：客服对话

```markdown
:::dialogue[客服咨询对话]
用户: 你好！我想了解一下会员服务
客服: 您好！很高兴为您服务。我们的会员服务包括...
用户: 价格是多少呢？
客服: 会员分为三个等级：基础版99元/年，专业版199元/年，企业版399元/年
用户: 好的，谢谢！
客服: 不客气！还有其他问题吗？
:::
```

### 示例2：教学问答

```markdown
:::dialogue[编程教学：什么是JavaScript？]
学生: 老师，什么是JavaScript？
老师: JavaScript是一种编程语言，主要用于网页开发
学生: 它和Java有什么区别吗？
老师: 它们是完全不同的语言。JavaScript主要用于前端，Java可以用于后端
学生: 明白了，谢谢老师！
老师: 不客气，继续加油学习！
:::
```

### 示例3：无标题对话

```markdown
:::dialogue
张三: 今天天气真不错！
李四: 是啊，阳光明媚的，适合出去走走
张三: 要不我们去公园吧？
李四: 好主意！半小时后见
:::
```

### 示例4：访谈记录

```markdown
:::dialogue[产品经理访谈]
记者: 作为产品经理，您认为最重要的能力是什么？
产品经理: 我认为是同理心，要站在用户角度思考问题
记者: 能具体说说吗？
产品经理: 比如我们要设计一个功能，不能只看技术可行性，更要考虑用户是否真的需要
记者: 这个观点很有启发性，谢谢分享！
:::
```

### 示例5：中文冒号

```markdown
:::dialogue
用户：请问如何开始使用这个插件？
助手：首先您需要安装插件，然后在设置中配置相关参数
用户：配置复杂吗？
助手：非常简单，按照向导提示几步就能完成
:::
```

## 视觉效果

### 设计特点

1. **微信风格**：左侧白色气泡，右侧绿色气泡
2. **角色头像**：首字母圆形头像，不同颜色渐变
3. **气泡箭头**：小三角箭头指示消息来源
4. **圆角设计**：现代化圆角矩形，视觉友好
5. **阴影效果**：柔和阴影，提升层次感
6. **响应式**：适配移动端和桌面端

### 消息布局

```
[头像] 白色气泡 ← 左侧消息
         右侧消息 → 白色气泡 [头像]
[头像] 白色气泡 ← 左侧消息
```

## 使用技巧

### 1. 角色命名建议

```markdown
// ✅ 推荐：简洁明了
用户: 内容
客服: 内容

// ✅ 推荐：英文也可以
User: content
Assistant: content

// ✅ 推荐：职位角色
产品经理: 内容
开发工程师: 内容
```

### 2. 对话长度

- 单条消息建议控制在100字以内
- 超长内容会自动换行
- 保持对话流畅性

### 3. 搭配使用

可以与其他 Markdown 语法组合：

```markdown
# 对话功能介绍

:::dialogue[功能演示]
用户: 这个对话功能真不错！
助手: 是的，支持很多实用场景

- 客服问答
- 教学演示
- 访谈记录

用户: 那太好了！我要用起来
:::
```

## 注意事项

### 语法规则

1. 每行对话必须包含冒号（`:` 或 `：`）
2. 角色名和内容之间至少需要一个空格
3. 空行会被自动忽略
4. 标题可选，建议添加以提高可读性

### 常见问题

**Q: 角色名可以使用特殊字符吗？**
A: 可以，但建议使用中英文、数字和常用符号

**Q: 可以有多少个角色？**
A: 没有限制，消息会自动左右交替显示

**Q: 支持表情符号吗？**
A: 支持，可以在对话内容中使用 emoji

**Q: 可以在微信公众号中使用吗？**
A: 完全可以，专门优化了微信显示效果
